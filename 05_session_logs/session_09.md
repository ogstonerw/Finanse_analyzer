# Session 09

## Что сделано

- Добавлена поддержка сущностей `news_items` и `events` в backend.
- Созданы миграции `007_create_news_items.sql` и `008_create_events.sql`.
- Реализован storage/repository слой для новостей и событий.
- В качестве прозрачного MVP-источника новостей выбран официальный раздел `Bank of Russia Monetary Policy Decisions`.
- Реализована базовая нормализация новости: `title`, `summary`, `body`, `source_id`, `published_at`, `collected_at`, `url`.
- Реализовано базовое событие поверх новости с полями `asset_id`, `event_type`, `summary`, `extracted_at`.
- Для MVP добавлена простая rule-based классификация событий по ключевым словам и тикерам.
- Добавлены read-only endpoint'ы `GET /api/v1/news`, `GET /api/v1/news/{id}`, `GET /api/v1/events`.
- Обновлен `README.md` backend под новый контур `news/events`.

## Архитектурные решения

- Новый collector вынесен в `internal/collectors` и использует официальный HTTP-источник без сложного парсинга большого числа сайтов.
- Связь события с активом выполняется явно и может отсутствовать: в этом случае `asset_id` сохраняется как `null`.
- Для MVP используется один event на одну нормализованную новость, что упрощает модель и не ломает текущую архитектуру.

## Ограничения текущего этапа

- Пока используется только один официальный источник новостей.
- AI-интеграция, sentiment scoring, importance scoring и сложный NLP не реализованы.
- Планировщик и периодическое обновление новостей пока не добавлены.
- Полноценная многообъектная event extraction логика пока не реализована.

## Результат

- Backend получил базовый контур новостей и событий, совместимый с текущей PostgreSQL-архитектурой и пригодный для дальнейшего расширения в сторону аналитики и прогнозного слоя.
